# Architecture

OrbitForge treats autonomous orbital operations as a governance and provenance problem.

## Inputs

- Asset class and orbit band
- Collision distance
- Fuel margin
- Crosslink health
- Override readiness
- Policy envelopes
- Event severity
- Policy triggers
- Handoff chains
- Resolution states

## Core idea

Orbital agents need more than telemetry. They need:

- visible governance states
- replayable maneuver provenance
- explicit handoff and override chains
- operator-readable containment guidance

## Outputs

- Orbital risk overview
- Event replay lane
- Incident board
- API payloads for mission ops and governance systems
